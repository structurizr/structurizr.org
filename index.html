Software architecture models as code

Structurizr builds upon "diagrams as code", allowing you to create multiple software architecture diagrams from a single model. There are a number of tools for creating Structurizr compatible workspaces, with the Structurizr DSL being the recommended option for most teams. This Structurizr DSL example creates two diagrams, based upon a single set of elements and relationships. 

workspace {

    model {
        user = person "User"
        softwareSystem = softwareSystem "Software System" {
            webapp = container "Web Application" {
                user -> this "Uses"
            }
            container "Database" {
                webapp -> this "Reads from and writes to"
            }
        }
    }

    views {
        systemContext softwareSystem {
            include *
            autolayout lr
        }

        container softwareSystem {
            include *
            autolayout lr
        }

        theme default
    }

}

Rendering tools

There are number of tools that can be used to render views in a Structurizr workspace, each offering a different set of features and integration options.


Structurizr cloud service, on-premises installation, and Lite
A browser-based rendering tool with diagrams that are zoomable, interactive, animatable, and embeddable. Markdown/AsciiDoc documentation and architecture decision records (ADRs) can also be published alongside the diagrams, while alternative visualisations (e.g. graphs and trees) allow you to explore your software architecture model.

Structurizr Site Generatr
Builds an interactive website based upon the diagrams and documentation container within a Structurizr workspace.
C4Viz
A web-based rendering tool that renders a zoomable set of C4-PlantUML diagrams.
Ilograph (via the Structurizr CLI)
A text-based modelling tool that provides an interactive way to navigate a hierarchical model. Ilograph doesn't natively support Structurizr or the C4 model, but the Structurizr CLI can be used to convert a Structurizr workspace to Ilograph format.
PlantUML (via the Structurizr CLI)
The Structurizr CLI can be used to export the views defined in a Structurizr workspace to PlantUML and C4-PlantUML formats.

Mermaid (via the Structurizr CLI)
The Structurizr CLI can be used to export the views defined in a Structurizr workspace to Mermaid format.
DOT/Graphviz (via the Structurizr CLI)
The Structurizr CLI can be used to export the views defined in a Structurizr workspace to DOT format for use with Graphviz.
D2 (via the Structurizr CLI)
The Structurizr CLI can be used to export the views defined in a Structurizr workspace to D2 format.
WebSequenceDiagrams (via the Structurizr CLI)
The Structurizr CLI can be used to export the dynamic views defined in a Structurizr workspace to WebSequenceDiagrams format.
Kroki
Kroki generates diagrams from a number of text-based formats, including the Structurizr DSL.
structurizr2csv
A tool to convert the views defined in a Structurizr DSL/JSON workspace to the diagrams.net CSV format.

Git for Confluence | Markdown, PlantUML, Graphviz, Mermaid
A Confluence plugin that will render a specific diagram from a Structurizr DSL file stored in your git repo.

Custom tools

Irrespective of how your define your workspace (e.g. using the Structurizr DSL), workspaces are transformed into an open JSON data format (OpenAPI 3.0 definition), so it's relatively straightforward to build your own custom tooling to consume that data; perhaps for rendering views with your own diagramming tool, or to integrate the data with your internal dashboards and service catalogs. Although JSON is an easy data format to work with, using something like Structurizr for Java will provide a quicker starting point. In this code snippet, the resulting Workspace object provides an easy way to navigate/manipulate/export the data: 


public static void main(String[] args) throws Exception {
    Workspace workspace = WorkspaceUtils.loadWorkspaceFromJson(new File("workspace.json"));
}

Getting started

The separation of workspace authoring from diagram rendering makes it possible to use the Structurizr tooling in a number of ways, but the easiest ways to get started are:

  1. Use the Structurizr DSL demo page to try the DSL and a number of diagram rendering options, without installing any tooling.
  2. Use the DSL in conjunction with Structurizr Lite; see Structurizr Lite - Quickstart for details.
